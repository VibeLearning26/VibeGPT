### 3. Document Upload Tests

```bash
TOKEN="your_admin_token_here"
HDR="Authorization: Bearer $TOKEN"

# Create test PDF
python -c "
import fitz
doc = fitz.open()
page = doc.new_page()
page.insert_text((72, 72), 'Test content')
doc.save('test.pdf')
doc.close()
"

# Upload with explicit source_type
curl -X POST http://localhost:8000/api/v1/admin/documents/upload \
  -H "$HDR" \
  -F "file=@test.pdf" \
  -F "subject_id=1ff740f4-d809-4e6a-896a-08b6c473e3be" \
  -F "source_type=pdf_notes"

# Upload with auto-detection (source_type=other)
curl -X POST http://localhost:8000/api/v1/admin/documents/upload \
  -H "$HDR" \
  -F "file=@test.pdf" \
  -F "subject_id=1ff740f4-d809-4e6a-896a-08b6c473e3be" \
  -F "source_type=other"
```

**Expected responses:**
- 201: `{id, document_name, status: "processing", source_type, file_size, message}`
- 409: `{"detail": "Document already exists"}` (duplicate)
- 413: `{"detail": "File exceeds maximum size"}` (>50MB)
- 400: `{"detail": "Unsupported file type: txt"}` (invalid type)
- 404: `{"detail": "Subject not found"}` (invalid subject_id)

---

### 4. Document Processing Verification

After upload, poll job status:
```bash
curl -H "$HDR" "http://localhost:8000/api/v1/admin/documents/{doc_id}/job"
```

Expected progression:
```
pending → running → completed
```
Final document status: `ready` with `total_chunks > 0`

---

### 5. Validation Tests

| Test | Command | Expected |
|------|---------|----------|
| Duplicate upload | Upload same file twice | 409 Conflict |
| Oversized file (>50MB) | `dd if=/dev/zero of=large.pdf bs=1M count=51` | 413 Payload Too Large |
| Invalid type (.txt) | Upload .txt file | 400 Unsupported file type |
| Missing subject | Use invalid UUID | 404 Subject not found |
| Missing module | Use invalid module UUID | 404 Module not found |
| Retry failed | POST /retry on failed doc | 200, job → pending |
| Publish ready doc | POST /publish on ready doc | 200, status=published |
| Archive published | POST /archive on published doc | 200, status=archived |

## Worker Verification

Terminal running worker should show:
```
Document worker started (poll_interval=10s, max_retries=3)
_claim_job: found job = <uuid>
Processing job <uuid> for document <uuid>
Job <uuid> completed: N chunks created
```

Or on failure:
```
Job <uuid> failed (retry 1/3): <error>
Job <uuid> failed permanently after 3 retries: <error>
```

## Troubleshooting

| Issue | Fix |
|-------|-----|
| API Connection refused | Start API: `python -m uvicorn app.main:app --port 8000` |
| Worker not processing | Start worker: `python -m app.workers.document_worker` |
| Frontend 404 on routes | Restart Next.js: `npm run dev` (port 3000/3001) |
| 401 Unauthorized | Re-login, check token in sessionStorage |
| 403 Forbidden | Check user role (admin vs student) |
| Job stuck in running | Check worker logs, restart worker |
| 0 chunks created | Check file content extractable, worker running |
